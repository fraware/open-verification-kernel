# FormalPR-Bench

FormalPR-Bench scores OVK behavior across deterministic lane fixtures plus extended categories that exercise routing, adversarial resilience, repair loops, and planner recall.

## Public artifacts

- `docs/benchmarks/leaderboard-badge.json`: shields.io endpoint payload used by the README badge.
- `docs/benchmarks/latest-leaderboard-summary.json`: trimmed leaderboard summary for docs and external tooling.
- `.verification/formal-pr-bench-leaderboard.json`: full leaderboard generated by `ovk bench` (uploaded in CI artifacts).

## Dimensions

- `cases_passed / cases_total`: aggregate correctness across all evaluated cases.
- `merge_decision_accuracy`: recommendation alignment (`allow`, `block`, `require_human_review`) against expected outcomes.
- `status_accuracy`: backend status alignment (`pass`, `fail`, `unknown`) against expected outcomes.
- `counterexample_usefulness`: whether emitted counterexamples or repair hints match expected classes.
- `backend_selection_accuracy`: routing correctness where a backend choice is evaluated.
- `evidence_honesty`: quality/evidence consistency checks, especially for adversarial fixtures.
- `intent_recall`: planner recall signal for expected intents on diff-driven cases.

## Category pass rates

`latest-leaderboard-summary.json` includes `category_pass_rates` extracted from leaderboard `summary.by_category` so external dashboards can track trends by category without parsing full case payloads.
